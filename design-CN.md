# 综述

mtmalloc 分为三级缓存：线程缓存，中心缓存，页堆。

每级缓存都使用哈希桶来管理内存，每个桶是一个自由链表。

# 线程缓存

线程缓存是每个线程独有一份的实例，访问无需加锁。

线程缓存的哈希桶是将要分配的大小映射到对应的自由链表。

线程缓存的自由链表会遵循一个慢启动的算法。

# 中心缓存

中心缓存是一个单例，负责均衡调度内存的分配和释放。

和线程缓存类似，中心缓存的哈希桶也是将要分配的大小映射到对应的自由链表。

不同的是，中心缓存使用的是一个叫做 span 的结构来管理连续页面的内存，中心缓存的每个自由链表都有一把自己的锁。

# 页堆

页堆是一个单例，负责向操作系统申请内存和处理外部内存碎片的问题。

与前两级不同，页堆的哈希桶是将内存页面的数量映射到对应的自由链表。

另外，页堆整体是受一把大锁保护的。

# 关于分配

当一个线程申请内存时，如果大小不超过 `thread_cache_max_bytes` (默认为 256KB) 将直接从线程缓存分配，并且无需加任何的锁。而大小超过 `thread_cache_max_bytes` 的内存申请，则会从页堆中分配。

具体来说，当线程缓存得到一个要分配的字节数时，会先通过哈希桶找到对应的自由链表。如果自由链表中有内存，则直接分配，否则会先从中心缓存中获取内存，存储到自由链表，再进行分配。中心缓存分配内存给线程缓存时，会执行一个慢启动算法，避免一次分配太多内存而导致浪费。

和线程缓存类似，当中心缓存从线程缓存那里得到一个要分配的字节数时，会先通过哈希桶找到对应的自由链表。不同的是，访问中心缓存的自由链表时需要对该自由链表加锁，防止数据竞争。另一个不同点在于，中心缓存的自由链表存储的是 span，在分配给线程缓存之前，span 会先被切分成合适的大小。当然，如果对应的自由链表为空，中心缓存会先从页堆中获取 span，存储到自由链表，再进行后续的分配。

至于页堆，页堆负责的是以页为最小单位向操作系统申请内存，并且将申请的内存页面管理到 span 结构中。span 是页堆的自由链表的元素，自由链表是通过页堆的哈希桶从内存页数映射过来的。访问页堆时，需要对页堆整体加锁。

# 关于释放

当一个线程释放内存时，如果这块内存是直接从页堆分配的，则会直接回收到页堆中，否则将会回到该线程的线程缓存中对应的自由链表下。当该自由链表中的内存过多，多余的内存会回收到中心缓存中，由中心缓存进行调度。

如果中心缓存中的某个自由链表下的某个 span 不再活跃，则该 span 会被回收到页堆中。

为了解决外部内存碎片的问题，回收到页堆的 span 会进行合并。只有当一个 span 大到无法管理时，该 span 才会被回收到操作系统中。因此，为了下一次的快速分配，mtmalloc 会尽可能将内存保留在缓存中。
